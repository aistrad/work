### 4.2 动态问题选择算法

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│                    动态问题选择引擎                                          │
│                                                                             │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  输入: final_scores[12], confidence, top_archetypes                         │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                                                                     │   │
│  │  function select_questions(scores, top2):                           │   │
│  │                                                                     │   │
│  │      questions = []                                                 │   │
│  │      score_gap = scores[top2[0]] - scores[top2[1]]                 │   │
│  │                                                                     │   │
│  │      # 第1题: 主原型特征验证                                        │   │
│  │      q1 = get_feature_question(top2[0])                            │   │
│  │      questions.append(q1)                                           │   │
│  │                                                                     │   │
│  │      # 第2-3题: 根据得分差距选择                                    │   │
│  │      if score_gap < 0.05:                                          │   │
│  │          # 差距小，需要差异场景题                                   │   │
│  │          q2 = get_differentiation_question(top2[0], top2[1])       │   │
│  │          q3 = get_differentiation_question(top2[0], top2[1])       │   │
│  │      else:                                                          │   │
│  │          # 差距大，继续验证主原型                                   │   │
│  │          q2 = get_feature_question(top2[0])                        │   │
│  │          q3 = get_shadow_question(top2[1])                         │   │
│  │                                                                     │   │
│  │      questions.append(q2, q3)                                       │   │
│  │                                                                     │   │
│  │      # 第4题: 副原型确认                                            │   │
│  │      q4 = get_shadow_question(top2[1])                             │   │
│  │      questions.append(q4)                                           │   │
│  │                                                                     │   │
│  │      return questions                                               │   │
│  │                                                                     │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  答案处理与得分调整:                                                        │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                                                                     │   │
│  │  答案          得分调整                    含义                      │   │
│  │  ───────────────────────────────────────────────────────────────   │   │
│  │  "太准了!"    archetype_score × 1.15     强烈确认                   │   │
│  │  "有一点"     archetype_score × 1.05     部分确认                   │   │
│  │  "不太对"     archetype_score × 0.85     否定，降低置信度           │   │
│  │                                                                     │   │
│  │  特殊处理:                                                          │   │
│  │  • 连续2次"不太对" → 触发备选原型计算                              │   │
│  │  • 全部"太准了" → confidence × 1.2                                 │   │
│  │                                                                     │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 一、Vibe OS的设计理念

### 1.1 从"画像"到"操作系统"的范式转变

传统的人格测试给用户一个静态的"画像"——你是什么类型的人。但人不是静止的，我们每天都在变化，我们的能量在流动，我们的状态在起伏。
