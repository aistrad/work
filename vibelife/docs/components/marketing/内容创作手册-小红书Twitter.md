## 二、小红书内容创作指南

### 2.1 内容类型

#### Type 1: 痛点共鸣笔记 (认知阶段)

**目标**: 让用户产生"说的就是我"的共鸣

**标题公式**:
```
[痛点] + [意外反转] + [emoji]

示例:
- "为什么你总是拖延?可能不是懒,是身份问题 🧠"
- "28岁还不知道自己想要什么,其实很正常 ✨"
- "工作3年没成长,不是你不行,是方法不对 💡"
```

**内容结构**:
```
[Hook 钩子] - 1 句话说中痛点
  ↓
[共鸣] - 3-5 个痛点场景描述
  ↓
[反转] - 揭示真正的原因
  ↓
[解决方案] - 简要介绍方法 (不硬推产品)
  ↓
[CTA 行动召唤] - 评论区互动
```

**示例**:

```
标题: 为什么你总是拖延?可能不是懒,是身份问题 🧠

[Hook]
拖延不是病,改不掉的才是病。

[共鸣]
• 明明知道该做,就是不想动
• 每次都要deadline前一天才开始
• 完成后又后悔"早点做多好"
• 发誓下次不拖,结果还是一样

[反转]
你以为是"懒",其实是身份冲突。

比如:
• 你想写作,但心里觉得"我不是作家"
• 你想健身,但心里觉得"我不是运动的人"
• 你想学英语,但心里觉得"我就是学不会"

你的行为,永远服务于你认为自己是谁。

[解决方案]
改变行为前,先改变身份。

Dan Koe 的方法:
1. 找到旧身份:"我是那种会拖延的人"
2. 设计新身份:"我是那种立即行动的人"
3. 用行为强化:"一个立即行动的人,会怎么做?"

[CTA]
评论区说说,你的旧身份是什么?👇

#拖延症 #自我成长 #人生教练 #DanKoe #身份转换
```

#### Type 2: 方法论科普笔记 (认知阶段)

**目标**: 建立专业性,传递价值

**标题公式**:
```
[时间/步骤] + [结果] + [方法名] + [emoji]

示例:
- "10分钟重置你的人生 | Dan Koe 快速重置法 🚀"
- "4步拆解年度目标 | Covey 大石头法 🪨"
- "6个问题找到人生方向 | 反愿景设计法 ✨"
```

**内容结构**:
```
[背景] - 为什么需要这个方法
  ↓
[核心理念] - 方法的底层逻辑
  ↓
[具体步骤] - 可执行的流程
  ↓
[案例] - 真实使用场景
  ↓
[工具推荐] - 自然引入 VibeLife
```

**示例**:

```
标题: 10分钟重置你的人生 | Dan Koe 快速重置法 🚀

[背景]
感觉迷茫、不知道方向?
试试 Dan Koe 的一日重置法。

硅谷创业者都在用,10分钟给人生装个导航。

[核心理念]
改变不是靠"努力",而是:
1. 看清你不想要的 (反愿景)
2. 描绘你想要的 (愿景)
3. 设计新的身份
4. 拆解成行动

[具体步骤]
第1步: 绝不清单 (5分钟后的自己)
"5年后如果什么都不改变,你最怕活成什么样?"

示例:
• 困在不喜欢的工作里
• 每天重复无意义的事
• 没有自己的时间

第2步: 愿景场景 (3年后的普通周二)
"3年后,理想生活的一个普通周二是什么样?"

示例:
• 早上9点在海边工作室写作
• 下午3点去接孩子放学
• 晚上有时间陪家人

第3步: 放弃旧身份
"要真正改变,你必须放弃什么身份?"

示例:
"我是那种会拖延的人" → 扔掉

第4步: 新身份宣言
"你想成为什么样的人?"

示例:
"我是那种立即行动的创作者"

第5步: 本周行动
"如果你已经是那个人,这周你会做什么?"

示例:
• 每天写作1小时
• 周三发布第一篇文章

[案例]
```

| 维度 | 倾向 | 示例 |
|-----|------|------|

### 4.2 文案 Dos & Don'ts

#### ✅ Dos (要做的)

```
✓ 用"你"而非"我们"
   ❌ "我们的产品能帮您..."
   ✅ "你可以用 VibeLife..."

✓ 用具体场景而非抽象概念
   ❌ "提升自我认知"
   ✅ "理解你为什么总是拖延"

✓ 用问句增强互动
   ❌ "很多人都迷茫"
   ✅ "你是不是也感觉迷茫?"

✓ 承认痛点,不回避
   ❌ "我们的产品很好"
   ✅ "我知道你很焦虑,但这不是你的问题"

✓ 提供可执行的建议
   ❌ "要更自律"
   ✅ "每天早上9点,做第一件最重要的事"
```

#### ❌ Don'ts (不要做的)

```
✗ 过度承诺
   ❌ "7天改变人生!"
   ✅ "10分钟找到方向,然后需要你持续行动"

✗ 贩卖焦虑
   ❌ "再不改变就完了!"
   ✅ "感觉迷茫很正常,我们一起找方向"

✗ 空洞鸡汤
   ❌ "相信自己,你一定行!"
   ✅ "你是食神格,天生适合内容创作"

✗ 硬推产品
   ❌ "快来下载 VibeLife!"
   ✅ "想试试这个方法?VibeLife 可以帮你"

✗ 贬低用户
   ❌ "你就是太懒了"
   ✅ "拖延可能不是懒,而是身份冲突"
```

| emoji | 多用 | 少用 |

| 视觉 | 图片/视频重要 | 文字为主 |

## 十、内容创作流程

### 10.1 标准流程

```
Step 1: 选题 (10分钟)
├─ 参考内容日历
├─ 关注热点话题
└─ 分析用户反馈

Step 2: 研究 (20分钟)
├─ 搜集相关资料
├─ 找3-5个参考案例
└─ 提炼核心观点

Step 3: 创作 (30分钟)
├─ 写标题 (5分钟,想3个选最好)
├─ 写开头 (10分钟,必须抓眼球)
├─ 写主体 (10分钟,结构清晰)
└─ 写结尾 (5分钟,有 CTA)

Step 4: 优化 (10分钟)
├─ 检查 Checklist
├─ 删除废话
├─ 增加互动点
└─ 配图/视频

Step 5: 发布 (5分钟)
├─ 选择最佳时间
├─ 添加标签
└─ 设置互动提醒

Step 6: 互动 (持续)
├─ 回复评论
├─ 点赞转发
└─ 收集反馈

Total: 约 75 分钟/篇
```

### 10.2 批量创作技巧

#### 同一主题,多平台复用

```
核心内容 (800字)
    │
    ├─ 小红书: 完整版笔记
    │   └─ 配图 + 标签
    │
    ├─ Twitter: 精简版 Thread
    │   └─ 拆成 5-8 条
    │
    └─ 视频号: 短视频脚本
        └─ 60秒浓缩版
```

#### 一篇内容,多次传播

```
周一: 发布完整笔记
周二: 提取金句,单独发
周三: 补充案例,再发
周四: 做成视频,再发
周五: 用户反馈,再发
```
