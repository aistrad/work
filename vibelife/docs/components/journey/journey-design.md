```
┌─────────────────────────────────────────────────────────────────────────┐
│                      Dan Koe 一日重置 → Journey 页面                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │

│   ═══════════════════════                                               │
│   • 心理考古：识别行为模式                                               │
│   • 卡点突破：找到阻碍                                                   │
│   → 在 Chat 中完成，结果存入 identity.patterns                          │

│   Phase 2: 设计 (Design)                                                │

│   • 反愿景：你最恐惧的未来                                               │
│   • 愿景：你真正想要的未来                                               │
│   • 身份：你需要成为谁                                                   │
│   → 在 Chat 中完成，结果展示在 Journey 顶部「北极星」                    │
│                                                                         │
│   Phase 3: 执行 (Execute)                                               │

│                                                                         │
│   Phase 4: 复盘 (Review)                                                │
│   ═══════════════════════                                               │
│   • 每日复盘 / 周复盘 / 月复盘                                           │
│   → 在 Chat 中完成，进度在 Journey 页面可视化                            │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 设计总结

```
┌─────────────────────────────────────────────────────────────────────────┐
│                       Journey 页面设计原则                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   1. Chat-First                                                         │
│      所有深度交互都回到 Chat，Journey 只负责「展示」和「勾选」            │
│                                                                         │
│   2. 状态驱动                                                            │

│                                                                         │
│   3. 层级清晰                                                            │
│      愿景 → 年 → 季 → 月 → 周 → 日，自上而下，一目了然                   │
│                                                                         │
│   4. 行动优先                                                            │
│      今日杠杆始终可见，降低执行摩擦                                       │
│                                                                         │
│   5. 快捷入口                                                            │
│      常用操作（调整、复盘、卡点）一键进入 Chat                           │
│                                                                         │
│   6. 渐进解锁                                                            │
│      未完成的区块置灰，引导用户按顺序完成设置                             │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```
