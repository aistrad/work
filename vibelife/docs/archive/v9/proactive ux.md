## 1. 核心框架应用

### 1.1 Jobs to Be Done 重新定义

用户不是来「看运势」或「签到」的，而是：
```
  ┌────────────┬────────────────────────┐
  │  表面需求  │       真正的 Job       │
  ├────────────┼────────────────────────┤
  │ 看八字运势 │ 获得对不确定性的掌控感 │
  ├────────────┼────────────────────────┤
  │ 签到打卡   │ 证明自己在改变         │
  ├────────────┼────────────────────────┤
  │ 合盘分析   │ 理解关系中的困惑       │
  ├────────────┼────────────────────────┤
  │ 聊天倾诉   │ 被真正理解和接纳       │
  └────────────┴────────────────────────┘
```
设计原则：每个功能都要围绕 Job，而非 Feature。

```
  ┌──────────────────────────────────────────────────────────────┐
  │  原设计                                                      │
  │  ────────────────────────────────────────────────────       │
  │  功能导向：「今日运势 · 事业有突破」                          │
  │                                                              │
  │  优化设计                                                    │
  │  ────────────────────────────────────────────────────       │
  │  Job 导向：「今天有个决定可能困扰你，我来帮你想清楚」        │
  │                                                              │
  │  心理学：Jobs to Be Done + Relevance（相关性）               │
  └──────────────────────────────────────────────────────────────┘
```

## 2. Onboarding 优化（基于 onboarding-cro）

### 2.1 Aha Moment 定义

VibeLife 的 Aha Moment：用户第一次感受到「它懂我」
```
  ┌──────┬──────────────────────────┬────────────┐
  │ 阶段 │           指标           │    目标    │
  ├──────┼──────────────────────────┼────────────┤
  │ 激活 │ 用户说「好准」或类似表达 │ 首次对话内 │
  ├──────┼──────────────────────────┼────────────┤
  │ 留存 │ 主动分享个人信息         │ 首周内     │
  ├──────┼──────────────────────────┼────────────┤

### 2.2 首次体验重构（Time-to-Value 优先）

当前问题：需要填写完整信息才能体验价值

优化方案：30 秒内给出第一个洞察

```
  ┌──────────────────────────────────────────────────────────────┐
  │  Step 1: 最小输入（5 秒）                                    │
  │  ────────────────────────────────────────────────────       │
  │  「输入你的出生日期，我告诉你一件有趣的事」                  │
  │                                                              │
  │  [1990] 年 [1] 月 [1] 日                   [告诉我]          │
  │                                                              │
  │  心理学：                                                    │
  │  • Activation Energy（降低启动能量）                        │
  │  • Curiosity Gap（好奇缺口）                                │
  │                                                              │
  ├──────────────────────────────────────────────────────────────┤
  │  Step 2: 立即价值（10 秒）                                   │
  │  ────────────────────────────────────────────────────       │
  │                                                              │
  │  ┌────────────────────────────────────────────┐             │
  │  │ 🌲 你是「甲木」型人格                       │             │
  │  │                                            │             │
  │  │ 甲木人像一棵大树：                          │             │
  │  │ • 有强烈的向上生长欲望                      │             │
  │  │ • 性格正直，有时显得固执                    │             │
  │  │ • 适合做领导者和开拓者                      │             │
  │  │                                            │             │
  │  │ 这像你吗？                                  │             │
  │  │                                            │             │
  │  │ [很像 👍] [有点像] [不太像]                │             │
  │  └────────────────────────────────────────────┘             │
  │                                                              │
  │  心理学：                                                    │
  │  • Barnum Effect（巴纳姆效应）                              │
  │  • Commitment & Consistency（点击「很像」= 小承诺）         │
  │  • Reciprocity（先给价值，再要信息）                        │
  │                                                              │
  ├──────────────────────────────────────────────────────────────┤
  │  Step 3: 深化引导（基于选择）                                │
  │  ────────────────────────────────────────────────────       │

  │  • Foot-in-the-Door（小承诺 → 大承诺）                      │
  │  • Investment（投入越多，越不想放弃）                       │
  │                                                              │
  └──────────────────────────────────────────────────────────────┘

- 起始进度 20%+（不从 0 开始，基于 Goal-Gradient）

### 3.4 Proactive Rule 文件优化

# 每日运势推送内容生成

## 文案策略

### 标题（Curiosity Gap）
- 制造好奇缺口，不要说完
- 暗示有「重要信息」
- 示例：
  - ❌ 「今日运势 · 事业顺利」
  - ✅ 「今天有件事要告诉你...」
  - ✅ 「发现一个今天的『窗口』」

### 正文（Loss Aversion + Scarcity）
## 1. 先说「机会」或「风险」（不说完）
## 2. 加入时间稀缺性（「下午 2-4 点」）
## 3. 暗示错过的后果

### 社交证明（Social Proof）
- 添加：「{count} 位 {daymaster} 同胞已查看」
- 使用真实的每日数据

### CTA（Single + Action-Oriented）
- 只保留一个 CTA
- 动词开头：「告诉我」「查看详情」「避开雷区」
