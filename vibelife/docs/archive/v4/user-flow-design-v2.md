### Stage 3: 深度访谈 → 3 个问题，LLM 个性化生成

**目标**: 通过访谈建立情感连接，收集个性化信息

**访谈流程**:

```
Q1: 固定问题 - 生辰信息（必需）
────────────────────────────────────────────────────────────────
八字: "你的出生时间？"（年月日时）
星座: "你的出生日期？"（年月日）

→ 系统计算命盘/星盘
→ LLM 根据命盘生成 Q2

Q2: LLM 个性化生成 - 选择题 + 可选文本追加
────────────────────────────────────────────────────────────────
示例（甲木日主）:
"作为甲木日主的你，在关系中是否常常感到：
 [ ] 需要扎根的安全感
 [ ] 向上生长的空间
 [ ] 被滋养和支持

想补充说说吗？（可选）___________"

→ LLM 根据 Q2 回答生成 Q3

Q3: LLM 追问 - 开放文本
────────────────────────────────────────────────────────────────
示例:
"你提到在感情中需要安全感。
 最近有什么事情让你感到不安全吗？"

___________________________________
```

**核心设计**:
- 问题要「触动人心」，让用户感到被理解
- 问题基于命盘特征个性化生成
- 逐步深入，第三个问题引导用户敞开心扉

**转化目标**: > 50% 开始访谈的用户完成

## 四、"来自未来的信" 详细设计

### 4.1 信的触发场景

| 场景 | 触发条件 | 内容侧重 |
|------|---------|---------|
| **新用户体验** | 完成深度访谈 | 性格洞察 + 当前困境共鸣 + 近期建议 |
| **生日** | 用户生日当天 | 年度回顾 + 新一年运势 + 生日祝福 |
| **新年** | 农历/公历新年 | 年度总结 + 新年展望 |
| **大运转换** | 大运交接期 | 阶段总结 + 新阶段预告 |
| **用户主动请求** | 点击"生成新信" | 当前状态 + 近期建议 |

### 4.2 信的内容生成逻辑

```python
def generate_future_letter(user, interview_answers, letter_type):
    """
    生成"来自未来的信"

    核心原则：
    1. 像写给朋友的信，有温度有洞察
    2. 不暴露数据来源（命盘/访谈/上传资料）
    3. 所有信息自然融入叙述
    4. 结尾引导付费（新用户）或提供价值（付费用户）
    """

    # 1. 收集所有数据源
    data = {
        "chart": user.chart,  # 命盘/星盘数据
        "interview": interview_answers,  # 访谈回答
        "context": user.life_context,  # 用户画像
        "letter_type": letter_type,  # 信的类型
    }

    # 2. 构建 Prompt
    prompt = f"""
    你是 Vibe，用户的命理/占星顾问。

    现在需要写一封"来自未来的信"给用户 {user.nickname}。

    用户数据：
    - 命盘：{data['chart']}
    - 访谈回答：{data['interview']}
    - 当前状态：{data['context']}

    写作要求：
    1. 第一人称叙事：用"我看到你..."、"我感受到..."
    2. 长度：10+ 句，详细阐述
    3. 结构：
       - 开头：直接的情感共鸣
       - 中间1：基于命盘的性格洞察
       - 中间2：基于访谈的当前共鸣
       - 结尾：来自未来的建议与祝福
    4. 绝对禁止提及技术细节
    5. 语气温暖但有力量

    输出格式：纯文本，段落分明
    """

    # 3. 生成信
    letter = llm.generate(prompt)

    return letter
```

### 4.3 信打动用户的关键点

**用户读到什么会触发购买？**

1. **"我被看见了"**
   - 信准确描述了用户内心的特质
   - "作为甲木日主的你，骨子里住着一棵树" — 精准的比喻
   - 用户感叹："这就是我！"

2. **"我被理解了"**
   - 信回应了访谈中提到的困境
   - "你提到感情中缺乏安全感，我完全理解"
   - 用户感到被共情，不是泛泛而谈

3. **"我有方向了"**
   - 信给出具体的、基于命盘的建议
   - "这段时间适合沉淀，不是行动"
   - 用户觉得有实际指导价值

4. **"我想看更多"**
   - 信末尾暗示完整报告有更多洞察
   - "这只是开始，人生K线图会展示更完整的画面"
   - 用户好奇心被激发

## 六、关键设计决策表

| 决策点 | 方案 | 理由 |
|--------|------|------|
| 入口体验 | 品牌沉浸式首页 (World Class 高级感) | 建立信任，差异化竞品 |
| 注册策略 | 访客模式优先，付费时再注册 | 降低门槛，减少流失 |
| 访谈设计 | 3个问题，LLM 实时个性化生成 | 建立情感连接，收集数据 |
| 信的形式 | 简体中文纸张风格，免费完整展示 | 先给价值，再引导付费 |
| 信的定位 | 付费功能 preview | 展示系统能力，激发购买 |
| 主动推送 | 进入主界面时推送到聊天 | 核心差异化，"主动关心你" |
| 离线合并 | 智能合并多日推送 | 避免信息过载 |
| 定价策略 | ¥19.9 买报告送首月订阅 | 低门槛，高价值感知 |
