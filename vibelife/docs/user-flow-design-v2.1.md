# VibeLife 用户流程设计报告 v2.1

> **文档版本**: v2.1
> **设计日期**: 2026-01-09
> **更新**: 基于深度访谈优化 Landing Page、人格选择、Insight Panel
> **核心目标**: 1000 付费用户，5% 转化率

---

## 访谈结论摘要 v2.1

| 关键决策点 | 确认结论 |
|-----------|---------|
| **Landing Page** | 单页表单，键盘输入生辰信息，关注点一起选 |
| **人格选择** | 3个精选（暖心闺蜜/毒舌损友/人生导师），访谈前选择 |
| **加载过程** | 计算步骤 + 命盘渐进绘制 + 特色功能预览 |
| **Insight Panel** | 今日要事+运势+情绪+待办+节点+快捷操作 |
| **信的开篇** | 悬念式，语气由人格决定 |
| **注册** | 按地区自动（大陆手机号/海外Google Apple） |
| **支付** | 全部展示（微信/支付宝/Stripe/PayPal） |
| **推送周期** | 每周回顾+展望，每月/季度深度报告 |

**人格称呼适配**（根据用户性别）:

| voice_mode | 男性用户 | 女性用户 | 保密/未填 |
|------------|---------|---------|----------|
| warm | 暖心好友 | 暖心闺蜜 | 暖心伙伴 |
| sarcastic | 毒舌损友 | 毒舌损友 | 毒舌损友 |
| wise | 人生导师 | 人生导师 | 人生导师 |

**设计要点**:
- 用图标展示人格特征
- 名称双语（中英文）
- 示例话术让用户预览风格
- 不显示情绪价值分数

**目标**: 建立情感连接，收集个性化信息，展示推理能力

```

│   │  │  ✓ 土克水，水生木受阻                                          │ │  │
│   │  │  → 你最近是否感到「想要成长但被压制」的感觉？                   │ │  │
│   │  └─────────────────────────────────────────────────────────────────┘ │  │
│   │                                                                       │  │
│   │  我注意到你的命盘中有一个特别的组合...                               │  │
│   │  想确认一下：                                                         │  │
│   │                                                                       │  │

│   │                                                                       │  │
│   │  ┌────────────────────────────────────────────────────────────┐      │  │
│   │  │ ○ 需要深深扎根的安全感                                     │      │  │
│   │  │ ○ 渴望自由向上生长的空间                                   │      │  │
│   │  │ ○ 想要被滋养和支持                                         │      │  │

│   │  └────────────────────────────────────────────────────────────┘      │  │
│   │                                                                       │  │
│   │  想补充说说吗？                                                       │  │
│   │  ┌────────────────────────────────────────────────────────────┐      │  │
│   │  │ 比如最近发生了什么让你有这种感觉...                        │      │  │
│   │  └────────────────────────────────────────────────────────────┘      │  │
│   │                                                                       │  │
│   └───────────────────────────────────────────────────────────────────────┘  │
│                                                                               │
│            ┌────────────────────────────────────────┐                        │
│            │           继续 →                       │                        │
│            └────────────────────────────────────────┘                        │
│                                                                               │
└──────────────────────────────────────────────────────────────────────────────┘
```

**访谈问题设计原则**:

1. **展示推理过程**：让用户看到 AI 是如何从命盘得出问题的
2. **不确定处请用户补充**：增加参与感，同时获取更精准信息
3. **选择题 + 开放补充**：降低门槛，但给想表达的用户空间
4. **问题触动人心**：不是泛泛的问题，而是基于命盘特征的精准提问

**Q2 追问示例**（基于用户选择生成）:

```
[温暖向导]

你选择了「需要深深扎根的安全感」...

我懂。作为甲木日主的你，虽然外表看起来独立坚强，
但内心其实很渴望有一个能让你安心停留的地方。

最近有什么事情让你感到「不安全」吗？
或者，是什么让你开始思考这个问题？

┌────────────────────────────────────────────────────────────┐
│                                                            │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

def merge_offline_pushes(user_id: str, days: int) -> str:
    """
    合并用户离线期间的推送

    规则：
    - 3天内：按时间倒序列出重点
    - 3-7天：合并成一段总结
    - 7天以上：只保留最重要的 2-3 条
    """
    pushes = get_missed_pushes(user_id, days)

    if days <= 3:

## 五、关键设计决策表（更新版）

| 决策点 | 方案 | 理由 |
|--------|------|------|
| Landing Page 表单 | 键盘输入，不用拨盘 | 更快，减少摩擦 |
| 关注点选择 | 可选，和生辰一起 | 降低门槛，但收集有用信息 |
| 加载过程 | 计算步骤 + 渐进绘制 + 功能预览 | 展示真实性 + 预览付费价值 |
| 人格选择 | 3个精选，访谈前选 | 增加参与感，差异化体验 |
| 人格名称 | 中英双语，设计感 | 不土，有品质感 |
| 访谈设计 | 展示推理过程 | 体现专业性，增加信任 |
| 信的开篇 | 悬念式 | 抓住注意力，增加阅读完成率 |
| 注册方式 | 按地区自动 | 符合用户习惯 |
| 支付方式 | 全部展示 | 给用户选择权 |
| Insight Panel | 及时更新的信息 | 每次打开都有新价值 |
| 推送周期 | 每日+每周+每月+季度 | 分层次，避免打扰 |
