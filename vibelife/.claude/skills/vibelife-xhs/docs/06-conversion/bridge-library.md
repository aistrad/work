## VibeLife 三大价值主张

### 1. 被深度理解

**核心表达**：
- "你值得被理解，不是被纠正"
- "有一个存在真正懂你"
- "不是告诉你该怎么做，是先理解你为什么这样"

**适用内容主题**：心理、情绪、焦虑、内耗、讨好型人格

**对应 Skill**：psych、lifecoach

---

### 2. 多维度看清自己

**核心表达**：
- "命理看天赋，心理看模式，占星看成长"
- "一个答案不够，你需要多个角度"
- "星座不是标签，是理解自己的入口"

**适用内容主题**：自我认知、性格分析、迷茫、方向、星座

**对应 Skill**：zodiac、bazi、jungastro

---

### 3. 长期陪伴成长

**核心表达**：
- "改变不是一天的事，我陪你走"
- "不是给你答案，是陪你找到答案"
- "24小时都在，不用等，不用预约"

**适用内容主题**：成长、方法论、自律、拖延、习惯

**对应 Skill**：lifecoach、psych

## Dan Koe 化 CTA 模板库

### 模式1：个人经历 + 邀请

**结构**：分享我的使用经历 → 描述变化 → 邀请你也试试

**模板**：
```
我第一次[使用/测试]的时候，发现[具体发现]。
不是坏事——是终于看清了，才有机会改变。

想看你的？评论"[关键词]"。
```

**示例**：
```
我第一次做认知扭曲测试的时候，发现自己有6项。
不是坏事——是终于看清了，才有机会改变。

想看你的？评论"测试"。
```

---

### 模式2：理念 + 行动

**结构**：一句有力量的理念 → 简单的下一步

**模板**：
```
*[核心理念金句]*

如果你也[状态/感受]，
评论"[关键词]"，[一句话说明获得什么]。
```

**示例**：
```
*真正的改变，从看清自己开始。*

如果你也想停止和自己打架，
评论"对话"，试试10分钟自我对话。
```

---

### 模式3：身份认同 + 入口

**结构**：点出读者身份/状态 → 给一个低门槛入口

**模板**：
```
如果你是[身份/状态]的人，
你可能需要的不是[常见解决方案]，而是[我们提供的]。

评论"[关键词]"，私发你入口。
```

**示例**：
```
如果你是凌晨三点还在想很多的人，
你需要的不是"别想了"，而是学会和想法相处。

评论"焦虑"，私发你练习入口。
```

---

### 模式4：问题 + 照见

**结构**：提出一个深刻的问题 → 邀请用工具找答案

**模板**：
```
问自己一个问题：
[一个能引发深度思考的问题]

如果你想找到答案，
评论"[关键词]"。
```

**示例**：
```
问自己一个问题：
你在害怕什么？

如果你想看清那个恐惧，
评论"对话"。
```

---

### 模式5：对比 + 选择

**结构**：对比旧方式和新方式 → 给出选择

**模板**：
```
你可以继续[旧方式]，
也可以试试[新方式]。

选择权在你。
想试试的，评论"[关键词]"。
```

**示例**：
```
你可以继续用"想太多"责备自己，
也可以试试学会和想法相处。

选择权在你。
想试试的，评论"整理"。
```

## 中间植入模板（用于第四段案例）

> 在核心内容的「举例」部分自然植入 VibeLife，让读者感受产品价值

### 自然植入三步法

1. **场景**：描述一个具体的人/时刻
2. **工具**：自然提到使用了什么（测试/看星盘/抽牌等）
3. **洞察**：具体发现了什么

### 模板A：用户故事

```
上次有个[身份]给我发消息，说[问题]。
我让TA试了[方法/工具]。
结果发现[具体洞察]。
TA说[反馈/变化]。
```

**示例**：
```
上次有个姑娘凌晨给我发消息，说又在想很多。
我让她做了个认知扭曲测试。
结果她发现自己最严重的是"读心术"——总是觉得自己知道别人在想什么。
她说意识到这点之后，焦虑轻了很多。
```

### 模板B：自己的经历

```
我有个习惯——每次[场景]，就会[使用工具]。
不是为了[否定的目的]，是[真实目的]。
上次[具体情况]，发现[洞察]。
知道这点后，[变化/好处]。
```

**示例**：
```
我有个习惯——每次情绪崩溃后，冷静下来会做一个自我对话练习。
不是为了证明自己对，是看看自己有没有掉进什么思维陷阱。
上次和人吵架后试了一下，发现自己"灾难化"特别严重。
知道这点后，下次就能早点发现自己在往最坏处想。
```

### 模板C：朋友/读者案例

```
有个[关系]以前觉得自己[问题]。
[看了/测了/试了]之后才明白：[洞察]。
TA说[反馈]。
```

**示例**：
```
有个读者说自己"什么都想做，什么都做不好"。
她看了自己的八字之后才明白：她是伤官配印格——天生就是多才多艺型。
她说这个视角让她松了一口气。
```

## 使用指南

### Step 1：确定内容主题

从映射表中找到对应的：
- 价值主张
- 对应 Skill
- CTA 方向

### Step 2：选择 CTA 模式

根据内容类型选择：
- 个人经历型：适合有真实使用案例的内容
- 理念型：适合认知升级类内容
- 身份认同型：适合痛点明确的内容
- 问题型：适合深度思考类内容
- 对比型：适合方法论类内容

### Step 3：组合生成

将理念桥梁 + CTA 模式 + Skill 专属话术组合：

```
[理念金句]

[1-2句个人化描述/邀请]

评论"[关键词]"，[获得什么]。
```

## 禁忌

### 不要做

- ❌ 纯功能描述："可以测试15种认知扭曲"
- ❌ 营销腔："限时免费"、"超值体验"
- ❌ 命令语气："快来试试"、"赶紧行动"
- ❌ 空洞承诺："改变你的人生"

### 要做到

- ✅ 有"我"的视角：分享自己的使用体验
- ✅ 有价值共鸣：先说理念，再说入口
- ✅ 有低门槛感："试试10分钟"、"看看你的"
- ✅ 有温度：像朋友推荐，不像销售推销
