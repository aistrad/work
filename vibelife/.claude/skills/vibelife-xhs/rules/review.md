## 单篇复盘流程

### Step 1：收集数据

询问用户：
```
请提供今日内容数据：
- 标题：
- 阅读量：
- 点赞数：
- 收藏数：
- 评论数：
- 分享数：（可选）
- 新增粉丝：（可选）

评论内容（文本或描述）：
```

### Step 2：计算指标

```
点赞率 = 点赞 / 阅读 × 100%
收藏率 = 收藏 / 阅读 × 100%
互动率 = (点赞 + 收藏 + 评论) / 阅读 × 100%
```

**基准**：
| 指标 | 一般 | 良好 | 优秀 |
|------|------|------|------|
| 点赞率 | <3% | 3-5% | >5% |
| 收藏率 | <2% | 2-4% | >4% |
| 互动率 | <5% | 5-10% | >10% |

### Step 3：分类评论

| 分类 | 特征 | 行动 |
|------|------|------|
| 看不懂 | "什么意思？" | 内容需更清晰 |
| 感兴趣 | "我也是！" | 记录热度 |
| 想更多 | "能讲讲XXX吗？" | **提取选题** |
| 指错误 | "这个不对吧" | 查证修正 |

### Step 4：四象限诊断

```
          评论质量高
              │
     Ⅱ        │        Ⅰ
  数据差评论好 │   数据好评论好
  （曝光问题） │   （优质内容）
              │
─────────────┼─────────────
              │
     Ⅲ        │        Ⅳ
  数据差评论差 │   数据好评论差
  （内容问题） │   （标题党）
              │
          评论质量低
```

**判断标准**：
- 数据好：互动率 ≥5%
- 评论好："感兴趣"+"想更多" ≥60%

**行动建议**：
| 象限 | 诊断 | 行动 |
|------|------|------|
| Ⅰ | 优质 | 做系列延展 |
| Ⅱ | 曝光问题 | 优化标题/封面 |
| Ⅲ | 内容问题 | 学习爆款 |
| Ⅳ | 标题党 | 提升价值 |

### Step 5：更新数据文件

**data/published.yaml** 追加：
```yaml
- id: "{date}-{pm/ev}"
  date: {date}
  time: {afternoon/evening}
  skill: {skill}
  title: "{title}"
  metrics:
    views: {n}
    likes: {n}
    saves: {n}
    comments: {n}
  rates:
    like_rate: "{n}%"
    save_rate: "{n}%"
    engagement_rate: "{n}%"
  quadrant: {I/II/III/IV}
  comment_mining: ["{选题1}", "{选题2}"]
  insights: ["{洞察}"]
```

**data/topic-pool.yaml** 追加（如有"想更多"选题）：
```yaml
high_priority:
  - skill: {skill}
    topic: "{选题}"
    source: "{date}评论"
    added_at: "{date}"
```

## 周度复盘流程

### Step 1：汇总数据

读取 `data/published.yaml` 本周记录，计算：
- 发布篇数
- 总阅读/平均阅读
- 平均互动率

### Step 2：找最佳/最差

- 最佳：互动率最高
- 最差：互动率最低
- 分析原因

### Step 3：Skill 分布

对比目标分布，调整下周计划：
```
psych: 2-3篇
zodiac: 2-3篇
tarot: 2-3篇
lifecoach: 1-2篇
bazi: 1-2篇
jungastro: 0-1篇
```

### Step 4：更新 data/insights.yaml

```yaml
successful_patterns:
  - pattern: "{成功模式}"
    week: "{week}"

failed_patterns:
  - pattern: "{失败模式}"
    lesson: "{教训}"
```
