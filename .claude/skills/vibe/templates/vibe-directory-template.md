# .vibe 目录模板

当用户启动新项目时，创建以下目录结构：

```
.vibe/
├── _index.md        # 项目摘要 + 当前状态
├── roadmap.md       # 阶段进度追踪
├── discovery.md     # Discovery 阶段产出
├── design.md        # Design 阶段产出
├── build.md         # Build 阶段产出
├── content.md       # Content 阶段产出
└── growth.md        # Growth 阶段产出
```

## 初始化脚本

当用户首次使用 /vibe 时，执行以下操作：

```bash
# 创建 .vibe 目录
mkdir -p .vibe

# 创建初始文件（使用上述模板）
# _index.md
# roadmap.md
# discovery.md
# design.md
# build.md
# content.md
# growth.md
```

## 状态更新规则

### 阶段状态值

```yaml
pending: 待开始
in_progress: 进行中
completed: 已完成
blocked: 阻塞
```

### 状态图标

```yaml
⏳: pending
🔄: in_progress
✅: completed
🚫: blocked
```

### 更新时机

1. **阶段开始** - 更新状态为 in_progress，记录开始时间
2. **阶段完成** - 更新状态为 completed，记录完成时间
3. **关键决策** - 更新 _index.md 的关键决策部分
4. **里程碑达成** - 更新 roadmap.md 的里程碑表格
