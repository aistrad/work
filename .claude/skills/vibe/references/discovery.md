# Discovery 阶段最佳实践

## 概述

Discovery 是项目的起点，目标是理解用户真实需求，确定产品方向和商业模式。

**核心原则**：用户需求是最重要的，需要不断刷新和对齐。

---

## 1. 用户需求采集

### 1.1 从社交媒体采集真实需求

**最佳实践流程**：

```
1. 确定目标平台
   ├── 小红书：产品体验、种草、吐槽
   ├── 微博：热点话题、情绪表达
   ├── 知乎：深度问题、解决方案探索
   ├── 抖音评论：即时反馈、情绪化表达
   └── 垂直社区：专业讨论、深度需求

2. 采集策略
   ├── 热帖分析：点赞数 > 1000 的帖子
   ├── 热门账号拆解：行业 KOL 的内容分析
   ├── 评论挖掘：热帖下的真实用户声音
   └── 关键词搜索：痛点相关关键词

3. 信息提取
   ├── 原始表达（用户原话）
   ├── 潜在需求（用户没说但想要的）
   ├── 痛点分级（频率 × 强度）
   └── 付费意愿信号
```

### 1.2 用户需求文档结构

```markdown
# 用户需求文档

## 1. 需求来源
| 平台 | 采集时间 | 样本数 |
|------|----------|--------|
| 小红书 | 2026-01-07 | 50 帖 |

## 2. 核心痛点
### 痛点 1: {{痛点描述}}
- 频率：高/中/低
- 强度：强烈/一般/轻微
- 原始表达：
  - "{{用户原话 1}}"
  - "{{用户原话 2}}"
- 潜在需求：{{分析}}

## 3. 用户期望
| 期望 | 优先级 | 现有方案 | 差距 |
|------|--------|----------|------|

## 4. 付费意愿信号
- {{信号 1}}
- {{信号 2}}

## 5. 关键洞察
- {{洞察 1}}
- {{洞察 2}}
```

### 1.3 访谈问题清单

**核心问题**（必问）：

1. **问题定义**
   - 这个项目要解决什么问题？
   - 用户的真实痛点是什么？（最好有数据支撑）
   - 这个问题有多痛？用户现在怎么忍受的？

2. **目标用户**
   - 你的目标用户是谁？
   - 他们在哪里？（线上/线下）
   - 他们现在怎么解决这个问题？

3. **需求验证**
   - 你观察到的用户原始需求是什么？
   - 有没有用户直接说过他们想要什么？
   - 付费意愿的信号是什么？

**深入问题**（根据回答追问）：

- 你是怎么发现这个需求的？
- 你和目标用户聊过吗？他们怎么说？
- 你觉得最让你惊讶的用户反馈是什么？

### 3.2 变现模式选择

| 模式 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| 订阅制 | 高频使用、持续价值 | 稳定收入、用户粘性 | 获客成本高 |
| 单次付费 | 明确价值、低频需求 | 转化简单 | 难以持续 |
| Freemium | 大众市场、病毒传播 | 规模化、数据积累 | 转化率低 |
| 增值服务 | 核心免费、高级付费 | 兼顾规模和变现 | 价值分层难 |

### 3.3 访谈问题清单

1. **北极星指标**
   - 北极星指标是什么？为什么选这个？
   - 这个指标如何衡量产品成功？

2. **变现路径**
   - 变现路径是什么？（订阅/单次付费/广告/增值服务）
   - 用户愿意为什么付费？

3. **定价策略**
   - 定价策略如何考虑？
   - 竞品的定价是多少？
   - 用户的心理价位是多少？

4. **PMF 验证**
   - PMF 如何验证？
   - 成功的标准是什么？
   - 失败了怎么办？

## 5. 常见陷阱

1. **解决方案先行**：先想好做什么再找需求
2. **假设当事实**：没有验证的需求假设
3. **忽视付费意愿**：用户说好但不付费
4. **竞品盲从**：照抄竞品而不思考差异
5. **北极星漂移**：指标不断变化
