     9→### 1.1 Jobs to Be Done 重新定义
    10→
    11→用户不是来「看运势」或「签到」的，而是：
    12→```
    13→  ┌────────────┬────────────────────────┐
    14→  │  表面需求  │       真正的 Job       │
    15→  ├────────────┼────────────────────────┤
    16→  │ 看八字运势 │ 获得对不确定性的掌控感 │
    17→  ├────────────┼────────────────────────┤
    18→  │ 签到打卡   │ 证明自己在改变         │
    19→  ├────────────┼────────────────────────┤
    20→  │ 合盘分析   │ 理解关系中的困惑       │
    21→  ├────────────┼────────────────────────┤
    22→  │ 聊天倾诉   │ 被真正理解和接纳       │
    23→  └────────────┴────────────────────────┘
    24→```
    25→设计原则：每个功能都要围绕 Job，而非 Feature。
    26→
    27→```
    28→  ┌──────────────────────────────────────────────────────────────┐
    29→  │  原设计                                                      │
    30→  │  ────────────────────────────────────────────────────       │
    31→  │  功能导向：「今日运势 · 事业有突破」                          │
    32→  │                                                              │
    33→  │  优化设计                                                    │
    34→  │  ────────────────────────────────────────────────────       │
    35→  │  Job 导向：「今天有个决定可能困扰你，我来帮你想清楚」        │
    36→  │                                                              │
    37→  │  心理学：Jobs to Be Done + Relevance（相关性）               │
    38→  └──────────────────────────────────────────────────────────────┘
    39→```

    81→  │                                                              │

   245→## 3. 暗示错过的后果

   247→### 社交证明（Social Proof）
