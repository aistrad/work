    81→### 2.2 VibeProfile：用户信息统一
    82→
    83→**职责**：一个人的完整画像
    84→
    85→```
    86→┌─────────────────────────────────────────────────────────────────┐
    87→│                       VibeProfile                                │
    88→├─────────────────────────────────────────────────────────────────┤
    89→│                                                                  │
    90→│  三层数据架构：                                                   │
    91→│  ─────────────────────────────────────────────────────────────   │
    92→│                                                                  │
    93→│  Layer 1: Identity（身份）                                       │
    94→│  ─────────────────────────────────────────────────────────────   │
    95→│  来源：用户填写                                                   │
    96→│  内容：birth_info, display_name, preferences                     │
    97→│                                                                  │
    98→│  Layer 2: Skills（Skill 原始数据）                               │
    99→│  ─────────────────────────────────────────────────────────────   │
   100→│  来源：各 Skill 实时写入                                         │
   101→│  内容：bazi.chart, zodiac.chart, lifecoach.{...}, tarot.{...}   │
   102→│                                                                  │
   103→│  Layer 3: Vibe（共享深度信息）← 核心创新                         │
   104→│  ─────────────────────────────────────────────────────────────   │
   105→│  来源：抽取（activities）+ 同步（skills）                         │
   106→│  内容：                                                           │
   107→│    • vibe.insight（我是谁：本质+动态+规律）                       │
   108→│    • vibe.target（我要成为谁：目标+聚焦+里程碑）                  │
   109→│                                                                  │
   110→│  ─────────────────────────────────────────────────────────────   │
   111→│                                                                  │
   112→│  核心原则：                                                       │
   113→│  • Skills 是原始数据：每个 Skill 存自己的数据，互不干扰           │
   114→│  • Vibe 是共享洞察：跨 Skill 共享，所有 Skill 都可读取           │
   115→│  • Vibe 双来源：抽取（activities）+ 同步（skills）               │
   116→│  • Single Source of Truth                                        │
   117→│                                                                  │
   118→└─────────────────────────────────────────────────────────────────┘
   119→```

   227→│                                                                  │
   228→└─────────────────────────────────────────────────────────────────┘
   229→```

   233→### 2.4 VibeTarget：我要成为谁
   234→
   235→**职责**：目标管理与成长陪伴
   236→
   237→**实现方式**：
   238→- **前端页面**：Journey 页面 (`/journey`)

   241→- **核心 Skill**：lifecoach（人生教练）协议流程生成结构化目标数据

   564→## 六、关键设计决策
   565→
   566→| 决策 | 内容 |
   567→|------|------|
   568→| Skill 分类 | 按领域划分（心理/命理/占星/直觉/职业/认知），不分层级 |
   569→| **VibeProfile 三层架构** | Identity + Skills + Vibe |
   570→| **Skills 是原始数据** | 每个 Skill 存自己的数据（bazi.chart, lifecoach.goals 等） |
   571→| **Vibe 是共享洞察** | vibe.insight（我是谁）+ vibe.target（我要成为谁） |
   572→| **Vibe 双来源** | 抽取（activities）+ 同步（skills） |
   573→| VibeInsight | 三维：本质(静态) + 动态(状态+趋势) + 规律(模式) |
   574→| VibeTarget | 三维：目标(goals) + 聚焦(focus) + 里程碑(milestones) |
   575→| 命理与目标 | 默认分离，用户主动询问时才融合 |
