     1→# Lifecoach V2 数据结构设计
     2→
     3→> Version: 2.1 | Date: 2026-01-19
     4→> 基于 Dan Koe + Stephen Covey 方法论融合
     5→> 支持"来自未来的你"角色扮演
     6→
     7→---
     8→
     9→## 1. 设计原则
    10→
    11→### 1.1 核心原则
    12→
    13→| 原则 | 说明 |
    14→|-----|------|
    15→| **方法论无关** | 数据结构不绑定具体方法论，Dan Koe、Covey、Ikigai 都用同一套结构 |
    16→| **多目标聚焦** | 北极星支持最多 3 个目标，与角色系统融合 |
    17→| **层级清晰** | 从北极星到每日行动，层层分解，路径明确 |
    18→| **角色平衡** | 支持 Covey 的多角色系统，追踪生活各领域平衡 |
    19→| **持续演进** | 支持渐进式填充，用户可以只做一部分 |
    20→| **自动拆解** | 用户设定愿景后，系统自动拆解路径，用户确认 |
    21→

   142→### 2.3 各部分详细定义
   143→
   144→#### 2.3.0 System (系统配置)
   145→
   146→Lifecoach 的系统级配置，包括方法论选择、角色设定、自动化选项。
   147→
   148→```json
   149→{
   150→  "system": {
   151→    "active_framework": "dankoe",
   152→    "persona": "future_self",
   153→    "auto_breakdown": true,
   154→    "daily_checkin": true,
   155→    "weekly_review": true,
   156→    "initialized_at": "2026-01-15T10:00:00Z"
   157→  }
   158→}
   159→```
   160→
   161→| 字段 | 类型 | 必填 | 说明 |
   162→|-----|------|-----|------|
   163→| active_framework | string | 是 | 当前使用的方法论 (dankoe/covey/okr)，用户同时只能选一套 |
   164→| persona | string | 是 | 系统角色 (future_self/coach)，见 1.2 语气与角色分层 |
   165→| auto_breakdown | boolean | 否 | 是否启用自动拆解（用户设定愿景后系统自动生成路径） |
   166→| daily_checkin | boolean | 否 | 是否启用每日签到提醒 |
   167→| weekly_review | boolean | 否 | 是否启用周复盘提醒 |
   168→| initialized_at | string | 否 | 首次初始化时间 |
   169→
   170→**方法论说明：**
   171→
   172→| 方法论 | 核心特点 | 适合人群 |
   173→|-------|---------|---------|
   174→| dankoe | 愿景/反愿景驱动，身份转换，每日杠杆 | 创作者、自由职业者 |
   175→| covey | 使命宣言，角色平衡，大石头 | 多角色平衡者、管理者 |
   176→| okr | 目标与关键结果，季度迭代 | 目标导向者、团队协作 |
   177→
   178→**自动拆解流程：**
   179→
   180→```
   181→用户设定愿景 → 系统自动生成路径 → 用户确认/调整 → 保存
   182→     │                │                  │
   183→     ▼                ▼                  ▼
   184→  north_star    roadmap (草稿)      roadmap (确认)
   185→```
   186→
   187→#### 2.3.1 North Star (北极星)
   188→
   189→终极方向，包含愿景、反愿景、使命、原则。当 `system.persona = future_self` 时，`vision_scene` 用于构建"来自未来的你"的口吻。
   190→
   191→```json
   192→{
   193→  "north_star": {
   194→    "vision": "一个通过创作影响百万人、时间自由、家庭幸福的人",
   195→    "vision_scene": "我现在坐在海边的工作室里，刚完成今天的写作。窗外是蓝色的大海，下午三点，我准备去接孩子放学...",
   196→    "vision_timeframe": "3年后",
   197→    "anti_vision": "一个为了安全感困在无意义工作中、与家人疏远的人",
   198→    "mission": "通过清晰的思考和写作，帮助人们设计自己的人生",
   199→    "principles": [
   200→      "健康第一",
   201→      "家庭优先于事业",
   202→      "诚实透明"
   203→    ],
   204→    "details": {
   205→      "anti_scene": "早上被闹钟惊醒，挤地铁上班，晚上加班到深夜..."
   206→    },
   207→    "source": "dankoe",
   208→    "created_at": "2026-01-15",
   209→    "updated_at": "2026-01-15"
   210→  }
   211→}
   212→```
   213→
   214→| 字段 | 类型 | 必填 | 说明 |
   215→|-----|------|-----|------|
   216→| vision | string | 是 | 一句话愿景 |
   217→| vision_scene | string | 否 | 愿景场景描述（用于 future_self 角色口吻） |
   218→| vision_timeframe | string | 否 | 愿景时间框架 (1年后/3年后/5年后) |
   219→| anti_vision | string | 否 | 一句话反愿景 |
   220→| mission | string | 否 | Covey 使命宣言 |
   221→| principles | string[] | 否 | 不可妥协的原则 |
   222→| details | object | 否 | 详细描述（方法论特有字段） |
   223→| source | string | 否 | 来源方法论 (dankoe/covey/okr) |
   224→
   225→**Future Self 角色说明：**
   226→
   227→当 `persona = future_self` 时，系统会基于 `vision_scene` 以"来自未来的你"的口吻与用户对话：
   228→
   229→```
   230→示例对话（warm + future_self）：
   231→"嘿，我是3年后的你。现在我正坐在海边工作室里，刚完成今天的写作。
   232→ 你现在做的每一个选择，都在把我们带向这里。今天的写作完成了吗？"
   233→
   234→示例对话（roast + future_self）：
   235→"我是3年后的你，正在海边喝咖啡。你呢？还在刷手机？
   236→ 醒醒，那个愿景不会自己实现。今天的杠杆行动，做了几个？"
   237→```
   238→
   239→#### 2.3.2 Goals (核心目标)
   240→
   241→用户自由设定的核心目标，最多 3 个。融合了角色与目标的概念。
   242→
   243→```json
   244→{
   245→  "goals": [
   246→    {
   247→      "id": "creative_freedom",
   248→      "name": "创作自由",
   249→      "description": "通过写作和创作实现时间与财务自由",
   250→      "year_target": "出版第一本书，建立1万订阅者",
   251→      "why": "这是我实现愿景的核心路径",
   252→      "priority": 1
   253→    },
   254→    {
   255→      "id": "family_happiness",
   256→      "name": "家庭幸福",
   257→      "description": "与家人建立深度连接",
   258→      "year_target": "每周3次高质量陪伴，每月1次家庭活动",
   259→      "why": "家人是我最重要的支持系统",
   260→      "priority": 2
   261→    },
   262→    {
   263→      "id": "health",
   264→      "name": "身心健康",
   265→      "description": "保持身体和心理的健康状态",
   266→      "year_target": "每周运动3次，保持冥想习惯",
   267→      "why": "健康是一切的基础",
   268→      "priority": 3
   269→    }
   270→  ]
   271→}
   272→```
   273→
   274→| 字段 | 类型 | 必填 | 说明 |
   275→|-----|------|-----|------|
   276→| id | string | 是 | 目标唯一标识 |
   277→| name | string | 是 | 目标名称（用户自由命名） |
   278→| description | string | 否 | 目标描述 |
   279→| year_target | string | 是 | 年度目标 |
   280→| why | string | 否 | 为什么这个目标重要 |
   281→| priority | number | 否 | 优先级 (1最高) |
   282→
   283→**约束：最多 3 个目标**，帮助用户聚焦。

   661→## 5. 使用流程
   662→
   663→### 5.1 首次使用流程
   664→
   665→```
   666→1. 北极星设定 (Phase 1)
   667→   ├── 选择方法论: Dan Koe / Covey / OKR
   668→   ├── 执行协议: 反愿景 → 愿景 → 原则
   669→   └── 写入: north_star
   670→
   671→2. 核心目标设定 (Phase 2)
   672→   ├── 设定核心目标 (最多3个)
   673→   ├── 每个目标设定年度目标
   674→   └── 写入: goals, roadmap.year
   675→
   676→3. 季度规划 (Phase 3)
   677→   ├── 确定季度聚焦
   678→   ├── 设定关键结果
   679→   └── 写入: roadmap.quarter
   680→
   681→4. 月度项目 (Phase 4)
   682→   ├── 确定本月 Boss
   683→   ├── 拆解任务
   684→   └── 写入: current.month
   685→
   686→5. 周计划 (Phase 5)
   687→   ├── 选择大石头 (按目标平衡)
   688→   └── 写入: current.week
   689→
   690→6. 每日执行 (Phase 6)
   691→   ├── 确认今日杠杆
   692→   └── 写入: current.daily
   693→```
   694→
   695→### 5.2 日常陪伴流程
   696→
   697→```
   698→每日早晨 (默认开启):
   699→├── 读取: current.daily, progress
   700→├── 推送: 今日杠杆提醒
   701→├── 用户响应后: 更新 progress.streak
   702→└── 晚间可选: 添加 journal (daily)
   703→
   704→每周日晚:
   705→├── 读取: current.week, goals
   706→├── 引导: 周复盘
   707→├── 添加: journal (weekly)
   708→└── 规划: 下周 rocks
   709→
   710→每月初:
   711→├── 读取: current.month, roadmap.quarter
   712→├── 引导: 月度复盘
   713→├── 添加: journal (monthly)
   714→└── 规划: 下月 project
   715→
   716→每季度初:
   717→├── 读取: roadmap, goals
   718→├── 引导: 季度复盘
   719→├── 添加: journal (quarterly)
