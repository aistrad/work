   211→├─────────────────────────────────────────────────────────────────┤

   214→│  ─────────────────────────────────────────────────────────────   │
   215→│  "你的创造者特质正在显化，近期频繁讨论写作与此呼应。"              │
   216→│                                                                  │
   217→│  展开：三维画像                                                   │
   218→│  ─────────────────────────────────────────────────────────────   │
   219→│  • 本质：我是什么（原型 + 特质）                                 │
   220→│  • 动态：我现在怎么样 + 往哪走（状态 + 趋势）                    │
   221→│  • 规律：我的模式（行为洞察）                                    │
   222→│                                                                  │
   223→│  ─────────────────────────────────────────────────────────────   │
   224→│                                                                  │
   225→│  核心原则：                                                       │
   226→│  • 本质是"倾向"，不是"定论"                                      │
   227→│  • 动态每天都在变，趋势展示成长方向                              │
   228→│  • 规律帮助自我觉察，而非预测命运                                │
   229→│                                                                  │
   230→└─────────────────────────────────────────────────────────────────┘
   231→```

   235→### 2.4 VibeTarget：我要成为谁
   236→
   237→**职责**：目标管理与成长陪伴
   238→
   239→**实现方式**：
   240→- **前端页面**：Journey 页面 (`/journey`)
   241→- **数据存储**：`skill_data.lifecoach`
   242→- **API 端点**：`/api/v1/skills/lifecoach/state_read` 和 `state_write`
   243→- **核心 Skill**：lifecoach（人生教练）协议流程生成结构化目标数据
   244→
   245→**VibeTarget 模型（与 VibeInsight 平行）**：
   246→
   247→```
   248→┌─────────────────────────────────────────────────────────────────┐

   255→│  │   ─────────────────────────────────────────────────────  │     │
   256→│  │                                                          │     │
   257→│  │   来源：对话抽取 + 专门协议流程                           │     │

   343→│                                                                  │
   344→│  数据来源：                                                       │
   345→│  • 专门协议：lifecoach 协议流程生成 north_star（优先级最高）      │

   349→│  • 用户手动：Journey 页面直接添加/编辑目标                        │
   350→│                                                                  │
   351→│  ─────────────────────────────────────────────────────────────   │
   352→│                                                                  │
   353→│  设计原则：                                                       │
   354→│  • 默认不加入命理元素（纯粹的目标管理）                          │
   355→│  • 用户主动询问时，才提供命理视角                                │
   356→│  • 重点是陪伴和鼓励，不是预测和判断                              │
   357→│                                                                  │
   358→└─────────────────────────────────────────────────────────────────┘

   560→## 六、关键设计决策
   561→
   562→| 决策 | 内容 |
   563→|------|------|
   564→| Skill 分类 | 按领域划分（心理/命理/占星/直觉/职业/认知），不分层级 |
   565→| VibeInsight | 三维：本质(静态) + 动态(状态+趋势) + 规律(模式) |
   566→| VibeTarget | **实现为 Journey 页面（skill_data.lifecoach）** |
   567→| VibeTarget 结构 | north_star + goals + current(week/daily/month) + progress + focus |
   568→| 数据来源 | VibeInsight 由 ProfileExtractor 生成；VibeTarget 由 lifecoach 协议生成 |
   569→| 命理与目标 | 默认分离，用户主动询问时才融合 |
   570→| 数据分层 | Identity → Skill Data (含 lifecoach) → VibeInsight（三层） |
