# Web Interface Guidelines: Core Principles & Best Practices

## Accessibility Foundation
"Icon-only buttons need `aria-label`" and form controls require labels or ARIA equivalents. Prioritize semantic HTML (`<button>`, `<a>`, `<label>`) over generic divs with click handlers. Interactive elements must support keyboard navigation via `onKeyDown`/`onKeyUp`.

## Visual & Focus Design
All interactive elements require visible focus states using `:focus-visible`. Never remove outlines without replacement. Maintain keyboard usability throughout the interface.

## Form Excellence
Inputs need correct `type` attributes, `autocomplete` values, and `name` attributes. Labels must be clickable via `htmlFor`. Disable paste-blocking; honor password managers. Show validation errors inline and focus the first error on submission.

## Performance & Scale
Virtualize lists exceeding 50 items. Prevent layout thrashing by batching DOM reads/writes. Use `min-w-0` on flex children to enable text truncation.

## Motion Respect
Honor `prefers-reduced-motion` preferences. Animate only `transform` and `opacity`. Avoid `transition: all`; list properties explicitly.

## Content Robustness
Handle empty states gracefully. Anticipate short and very long user inputs. Use truncation or line clamping for overflow text.

## Internationalization
Replace hardcoded date/number formats with `Intl.DateTimeFormat` and `Intl.NumberFormat`. Detect language via headers, not IP geolocation.

## Anti-patterns to Flag
Inline `onClick` navigation, `user-scalable=no`, `outline-none` without focus replacement, images without dimensions, unvirtualized large arrays.
