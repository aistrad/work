     1→# VibeLife 主页面 UI/UX 设计规范
     2→
     3→> **Version**: 2.0
     4→> **Date**: 2026-01-05
     5→> **Author**: AI Design Assistant (frontend-design)
     6→> **References**: `vibelife spec v2.0/v2.1` + `fortune_ai/assets` + Mentis (Vibe Diary)
     7→
     8→## 设计参考与风格定位
     9→
    10→**产品定位**: The Living Companion Ecosystem（多 Skill 独立站 + Vibe ID）
    11→**用户心智**: 先体验后注册；一次输入 → 立即获得“我被看见”的感觉
    12→**品牌气质**: 沉稳智慧、温暖可信、不过度神秘、不说教
    13→**视觉方向**: VELLUM OS / LIVING CONSTELLATION（羊皮纸质感 + 墨色排版 + 玻璃胶囊）
    14→**参考**:
    15→- `fortune_ai/assets`: editorial serif、柔雾背景、frosted capsule、soft blob（氛围与节奏参考，不做视觉复刻）
    16→- Mentis: warm paper + zen spacing（信息密度与留白参考）
    17→**避免**:
    18→- 大面积霓虹/紫色渐变、廉价星空纹理、满屏 emoji
    19→- 过度 3D 球体炫技 / 交互沉浸（Soul Orb 式），宁可“轻、慢、稳”
    20→
    21→## 0.1 签名视觉语言 (Signature Language)
    22→
    23→1. **Vibe Capsule**: frosted glass 的输入胶囊，是所有站点的“第一触点”（Landing + Chat Omnibar）
    24→2. **Vibe Core**: 柔和的有机形（非炫技 3D），作为 AI 伴侣的“情绪锚点”（Hero/Avatar/Loading）
    25→3. **Constellation Thread**: 极细线 + 点阵，把技能与洞察“串起来”（分隔线/背景纹理/进度提示）
    26→4. **Insight Seal**: 小型金色“印章/戳记”，给 Insight Card 统一的仪式感（可保存/可分享）
    27→
    28→## 0.2 快速落地（工程清单）
    29→
    30→> 目标：先把“底子”统一，再做每个 Skill 的差异化细节。
    31→
    32→**P0（当天可做）**
    33→- 字体替换：移除 Inter；在 `vibelife/apps/web/src/app/layout.tsx` 使用 `next/font/google` 引入 `Fraunces` + `Bricolage Grotesque`（中文回退 `Noto Sans/Serif SC`），并同步 `vibelife/apps/web/tailwind.config.ts` 的 `fontFamily`。
    34→- 全局底色：在 `vibelife/apps/web/src/app/globals.css` 引入 `.vellum-bg`（或 `body` 级别）背景；去掉大面积彩色渐变底。
    35→- CTA 统一：Primary CTA 用 `Ink`（黑/墨色），Gold 只做点缀；避免 “彩虹渐变按钮”。
    36→- Icon 统一：emoji 仅做 wireframe 占位；落地用 `lucide-react`（线性图标同一笔触/同一尺寸）。
    37→
    38→**P1（1-2 天）**
    39→- Tokens 化：把颜色/圆角/阴影写成 CSS variables（light/dark），在 `vibelife/apps/web/tailwind.config.ts` 里只引用变量（避免散落 hex）。
    40→- Skill 变体：根据 `NEXT_PUBLIC_SITE_ID` 注入 `--skill-primary/--skill-secondary`，做到“一套组件，多站点皮肤”。
    41→- Chat/Insight 组件对齐：统一 message bubble、omnibar、insight card 的边界/投影/间距（减少站点间割裂感）。
    42→
    43→**P2（可迭代）**
    44→- Shareable Cards：按 `v2.1` 尺寸规格输出（1080×1920/1080×1080/1200×630），把“分享”做成增长入口。
    45→- Vibe Core 资产：给每个 Skill 一套静态（或极轻微动）“Vibe Core” SVG/PNG，避免重 3D 依赖与首屏性能风险。

   109→## 1.2 字体系统
   110→
   111→```
   112→Display Serif (情感标题/英文大标题):
   113→  font-family: 'Fraunces', 'Noto Serif SC', 'Source Han Serif SC', serif
   114→  用于：Hero 标题、Insight 标题、关键短句（更“编辑部”，更克制）

   176→/* 轻量玻璃 */

   180→}
   181→```
   182→
   183→### 纸质底纹（Grain）+ 星点纹理（Constellation）
   184→
   185→> 目标：让大面积留白“有空气感”，但不抢内容；所有纹理都应在 1 秒内被用户忽略。
   186→
   187→```css
   188→/* 背景纹理建议：用伪元素叠加，避免污染业务组件 */
   189→.vellum-bg {
   190→  position: relative;
   191→  background: #fbf7f1;
   192→}
   193→
   194→.vellum-bg::before {
   195→  content: "";
   196→  position: absolute;
   197→  inset: 0;
   198→  pointer-events: none;
   199→  opacity: 0.55;
   200→  background:
   201→    radial-gradient(circle at 20% 10%, rgba(184, 138, 68, 0.10), transparent 45%),
   202→    radial-gradient(circle at 80% 30%, rgba(60, 58, 122, 0.08), transparent 50%),
   203→    radial-gradient(circle at 50% 70%, rgba(15, 118, 110, 0.06), transparent 55%),
   204→    radial-gradient(rgba(28, 26, 23, 0.06) 1px, transparent 1px);
   205→  background-size: auto, auto, auto, 28px 28px;
   206→  mix-blend-mode: multiply;
   207→}
   208→```

   210→## 1.5 动画系统
   211→
   212→### 入场动画
   213→```
   214→fade-in:      0.4s ease-out, opacity 0→1
   215→fade-in-up:   0.4s ease-out, opacity 0→1, translateY 20px→0
   216→scale-in:     0.3s ease-out, scale 0.95→1, opacity 0→1
   217→slide-right:  0.3s ease-out, translateX -20px→0, opacity 0→1
   218→```
   219→
   220→### 缓动函数
   221→```
   222→ease-out-expo:  cubic-bezier(0.16, 1, 0.3, 1)    - 高端流畅感
   223→ease-spring:    cubic-bezier(0.34, 1.56, 0.64, 1) - 弹性活泼
   224→```
   225→
   226→### 微交互
   227→```
   228→hover:   scale 1.02-1.05, 0.2s
   229→active:  scale 0.98, 0.1s
   230→focus:   ring-2 ring-accent/20, 0.15s
   231→```
