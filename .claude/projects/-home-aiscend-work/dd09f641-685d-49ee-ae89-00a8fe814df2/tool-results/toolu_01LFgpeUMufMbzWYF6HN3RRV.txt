    35→## 1. 设计哲学（不变）
    36→
    37→### 1.1 核心假设
    38→
    39→人类心理痛苦的根源在于**自我-自性轴线的断裂**（Ego-Self Axis Alienation）。
    40→
    41→现代人面临三重困境：
    42→1. **意义断裂**：事件发生但没有可承载的解释框架
    43→2. **效能断裂**：解释即便成立，也没有下一步行动
    44→3. **身份碎片化**：社交媒体塑造的多重"表演性自我"导致核心自我感的丧失
    45→
    46→### 1.2 OS 的一句话定义
    47→
    48→**Soul OS = 一个把"情绪—解释—行动—反馈"外部化的自我调节系统**
    49→
    50→### 1.3 北极星指标（WCG）
    51→
    52→**Weekly Closed-loop Guidance**：用户在一周内完成 ≥1 次"触发 → 指引 → 行动 → 反馈"闭环。
    53→
    54→```
    55→完整闭环 = Checkin/Chat → Insight/Prescription → Commitment → Action → Reflection → Memory Update

   259→## 5. 优化：anti-dependency 策略
   260→
   261→### 5.1 问题分析
   262→
   263→当前 `daily_limit` 触发过于激进，导致：
   264→- 新用户首次对话即被截断
   265→- 无法获得目标设定与行动处方
   266→- 用户体验断裂
   267→
   268→### 5.2 优化后的触发规则
   269→
   270→```python
   271→def check_anti_dependency(user_id: int, session_id: str) -> Optional[Dict]:
   272→    """
   273→    优化后的 anti-dependency 检测
   274→    """
   275→    # 获取用户上下文
   276→    user = get_user(user_id)
   277→    today_sessions = count_today_sessions(user_id)
   278→    today_messages = count_today_messages(user_id)
   279→    has_active_commitment = has_pending_commitment(user_id)
   280→    days_since_register = (datetime.now() - user.created_at).days
   281→    has_completed_onboarding = user.onboarding_completed
   282→
   283→    # 新用户豁免期（注册7天内）
   284→    if days_since_register < 7 and not has_completed_onboarding:
   285→        return None  # 不触发 anti-dependency
   286→
   287→    # 当日对话轮数阈值（从5提升到8）
   288→    DAILY_MESSAGE_LIMIT = 8
   289→
   290→    # 有未完成任务时，降低阈值
   291→    if has_active_commitment:
   292→        DAILY_MESSAGE_LIMIT = 5
   293→
   294→    # 触发条件
   295→    if today_messages >= DAILY_MESSAGE_LIMIT:
   296→        # 即使触发，也必须返回至少1个可执行任务
   297→        suggested_task = get_or_create_minimal_task(user_id)
   298→        return {
   299→            "triggered": True,
   300→            "intervention_type": "daily_limit",
   301→            "message": _build_intervention_message(suggested_task),
   302→            "suggested_tasks": [suggested_task]  # 必须有任务
   303→        }
   304→
   305→    return None
   306→
   307→def _build_intervention_message(task: Dict) -> str:
   308→    """
   309→    构建干预消息，必须包含可执行任务
   310→    """
   311→    return f"""今天我们聊了很多，信息已经足够了。
   312→
   313→现在最重要的是**先行动**：
   314→> {task['title']}（约{task.get('minutes', 5)}分钟）
   315→
   316→完成后我们再继续。"""
   317→```
   318→
   319→### 5.3 触发条件对比
   320→
   321→| 条件 | 优化前 | 优化后 |
   322→|-----|-------|-------|
   323→| 日消息阈值 | 5 | 8（有任务时5） |
   324→| 新用户豁免 | 无 | 注册7天内豁免 |
   325→| 返回任务 | 可选 | **必须返回至少1个任务** |
   326→| 首次会话 | 可能触发 | 豁免 |

   330→## 6. 优化：A2UI 强制渲染
   331→
   332→### 6.1 问题分析
   333→
   334→后端返回 A2UI 的 `action_buttons`，但前端未渲染，导致：
   335→- 用户无法点击"开始行动"等按钮
   336→- OS 停留在文字建议层
   337→
   338→### 6.2 A2UI 渲染规范（前端必须实现）
   339→
   340→```typescript
   341→interface A2UIComponent {
   342→  type: 'markdown_text' | 'action_buttons' | 'chart' | 'form' | 'card';
   343→  title?: string;
   344→  data: any;
   345→}
   346→
   347→interface ActionButton {
   348→  label: string;
   349→  action: {
   350→    type: 'start_task' | 'schedule_task' | 'open_panel' | 'opt_out' | 'checkin';
   351→    task_id?: string;
   352→    panel?: string;
   353→  };
   354→}
   355→
   356→// 前端必须实现的 action 处理映射
   357→const actionHandlers = {
   358→  'start_task': async (action) => {
   359→    await api.post('/api/dashboard/task/accept', {
   360→      task_id: action.task_id,
   361→      commitment_type: 'start_task'
   362→    });
   363→    refreshTasks();
   364→  },
   365→  'schedule_task': async (action) => {
   366→    await api.post('/api/dashboard/task/accept', {
   367→      task_id: action.task_id,
   368→      commitment_type: 'schedule_task'
   369→    });
   370→    refreshTasks();
   371→  },
   372→  'open_panel': (action) => {
   373→    const panelMap = {
   374→      'quests': 'tasks',  // 统一命名
   375→      'tasks': 'tasks',
   376→      'status': 'status',
   377→      'trends': 'trends'
   378→    };
   379→    setActiveTab(panelMap[action.panel] || action.panel);
   380→  },
   381→  'opt_out': async () => {
   382→    // 记录用户选择暂不执行
   383→    toast('好的，有需要随时告诉我');
   384→  },
   385→  'checkin': () => {
   386→    openCheckinModal();
   387→  }
   388→};
   389→```

   576→## 12. 验收标准（WCG + 新增）
   577→
   578→### 12.1 原有 WCG 验收
   579→
   580→```sql
   581→WITH weekly_loop AS (
   582→  SELECT
   583→    user_id,
   584→    DATE_TRUNC('week', created_at) AS week,
   585→    BOOL_OR(accepted_at IS NOT NULL) AS has_commitment,
   586→    BOOL_OR(status = 'done') AS has_action
   587→  FROM fortune_commitment
   588→  GROUP BY user_id, DATE_TRUNC('week', created_at)
   589→)
   590→SELECT
   591→  user_id,
   592→  week,
   593→  (has_commitment AND has_action) AS wcg_achieved
   594→FROM weekly_loop;
   595→```
