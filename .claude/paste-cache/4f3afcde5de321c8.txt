问题 1: Skill 发现与业务模型                                                                                                                 
                                                                                                                                               
  当前 VibeLife 的 Skill 有两种类型：                                                                                                          
  - 始终激活型 (core, mindfulness) - 基础能力，不需要单独订阅    - core,mindfuleness, lifecoach                                                                              
  - 专业型 (bazi, zodiac, tarot, jungastro, career) - 需要出生信息/计算                                                                        
                                                                                                                                               
  您期望的商业模型是：                                                                                                                         
  1. A) 全包订阅 - 用户订阅 Premium 后解锁所有 Skill                                                                                           
  2. B) 按 Skill 订阅 - 每个 Skill 独立定价，用户按需购买                                                                                      
  3. C) 混合模式 - 基础 Skill 免费，高级 Skill 需单独订阅或消耗次数      - yes，core，mindfulnes和lifecoach  默认，且免费。但除了core，其他所有skill都可以unsubscribe                                                                    
                                                                                                                                               
  这个决定会直接影响 Skill 管理系统的设计。                                                                                                    
                                                                                                                                               
  问题 2: Skill 发现的入口设计                                                                                                                 
                                                                                                                                               
  您希望用户在哪里发现新 Skill？        all                                                                                                       
  1. A) 专门的 Skill 市场页面 - 类似 App Store，独立浏览                                                                                       
  2. B) 对话中的智能推荐 - AI 根据对话自动推荐合适的 Skill                                                                                     
  3. C) 设置页面的功能管理 - 融入现有设置页                                                                                                    
  4. D) 首页卡片推荐 - 类似今日运势的卡片展示                                                                                                  
                                                                                                                                               
  问题 3: 订阅/取消订阅的粒度                                                                                                                  
                                                                                                                                               
  用户应该能管理到什么粒度？                                                                                                                   
  1. A) Skill 级别 - 订阅/取消整个 Skill（如：取消八字）       - yes                                                                                
  2. B) 场景级别 - 订阅 Skill 下的特定服务（如：只要八字的"每日运势"，不要"完整报告"）                                                         
  3. C) 提醒级别 - 精细控制每种推送（如：只要"流月提醒"，不要"节气提醒"）                                                                      
                                                                                                                                               
  问题 4: 与主动模块的集成                                                                                                                     
                                                                                                                                               
  现有 ProactiveEngine 已支持 Skill 级 reminders.yaml。您期望：                                                                                
  1. A) 简单开关 - 用户只能整体开/关某个 Skill 的推送                - yes                                                                          
  2. B) 分类管理 - 按提醒类型分类（运势类、事件类、目标类），用户选择想要的类别                                                                
  3. C) 完全自定义 - 用户可以精细到每个提醒类型的开关
