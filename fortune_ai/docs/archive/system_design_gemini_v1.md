+## 1. 核心战略与价值主张 (Strategic Core)
+
+### 1.1 市场空白与机会点
+基于 `survey social.md` 与 `survey real expert.md` 的深度交叉分析，我们发现当前市场存在显著的结构性断层：
+*   **竞品不足（The Gap）**：
+    *   **传统APP（如测测、灵机）**：算法专业但体验陈旧，充斥着恐吓式营销，缺乏美感与情感连接。
+    *   **西方竞品（Co-Star, The Pattern）**：审美与心理叙事极佳，但缺乏东方命理（八字/紫微）的底层逻辑支撑，且无法满足中国用户对“改运/仪式”的功利性需求。
+    *   **AI算命现状**：目前的AI多为“查表式”或“随机生成”，缺乏逻辑自洽性，且存在严重的“幻觉”，无法建立类似真人大师的信任感。
+*   **机会点（The Opportunity）**：
+    *   **从“被动查询”到“主动干预”**：用户不再满足于“会发生什么”，而是需要“现在该做什么”。
+    *   **“赛博仪式”的合法化**：电子木鱼、好运壁纸的流行证明，用户需要低门槛的、数字化的心理抚慰工具。
+    *   **可视化与审美红利**：利用 GenAI (Nano Banana) 技术，将枯燥的命理数据转化为高传播性的视觉图腾。
+
+### 1.2 核心价值主张 (UVRA)
+**“不仅仅是预测未来的算命工具，而是基于东方智慧与AI推理的人生风险管理系统 (Life-Risk Management OS)。”**
+
+*   **Why us?** 我们通过 **“混合RAG架构（确定性算法+知识图谱+LLM）”** 解决了AI胡说八道的问题，同时通过 **“主动服务系统”** 提供了比真人大师更即时、更全天候的陪伴。
+
+### 1.3 北极星指标 (North Star Metric)
+**每日有效指引交互数 (Daily Active Guidance Interactions, DAGI)**
+*定义*：用户不仅打开APP，且完成了至少一次“闭环行为”（如：点击了主动推送的运势建议、完成了3分钟电子木鱼仪式、或生成并保存了今日能量壁纸）。这代表了用户将系统作为生活决策或情绪调节的一部分。

+## 2. 用户画像与需求洞察 (Persona & Needs)
+
+### 2.1 核心用户画像：The Anxious Optimizer (焦虑的优化者)
+*   **典型特征**：22-35岁，一二线城市，受过良好教育，面临职场内卷与情感不确定性。
+*   **心理侧写**：他们既相信科学，又迷恋玄学（实用主义灵性）。他们使用玄学不是为了迷信，而是为了**“归因”**（不是我无能，是水逆）和**“确认”**（获得行动的勇气）。
+
+### 2.2 痛点-痒点-爽点分析
+*   **痛点 (Pain - 必须解决)**：
+    *   **决策瘫痪**：在面对复杂选项（跳槽、分手）时，缺乏客观的第三方参考系。（*来源：survey real expert.md 中提到的“外脑”需求*）
+    *   **情绪无助**：深夜焦虑时，真人大师不在线，通用AI太冰冷。（*来源：survey social.md 中提到的“陪伴”需求*）
+*   **痒点 (Itch - 潜意识欲望)**：
+    *   **身份构建 (Identity)**：渴望被贴上独特、深刻的标签（如“伤官生财格”），并以此作为社交货币。（*来源：Co-Star 的成功逻辑*）
+    *   **窥私欲 (Voyeurism)**：想了解暗恋对象或讨厌同事的真实性格，但无法直接询问。（*来源：survey other product.md 中的“幽灵档案”*）
+*   **爽点 (Delight - 超预期)**：
+    *   **视觉显化**：每天早上自动生成一张基于当日运势的、极具艺术感的“能量壁纸”，直接击中审美。（*来源：Nano Banana 视觉技术应用*）
+    *   **全知视角**：系统主动提示“今天你老板火气大，建议避开”，仿佛拥有上帝视角。（*来源：concenpt design.md 中的“宇宙瞭望塔”*）
+
+### 2.3 用户旅程地图 (User Journey: The "Digital Sanctuary" Loop)
+1.  **触发 (Trigger)**：系统后台监测到“水星逆行”进入用户命盘 -> 推送通知：“⚠️ 沟通预警：今日不宜发送重要邮件”。
+2.  **行动 (Action)**：用户点击通知进入APP -> 看到“全景时间轴”上的能量低谷。
+3.  **投入 (Investment)**：用户询问AI：“那我该怎么回复老板？” -> AI基于“人情世故”模型生成话术。
+4.  **酬赏 (Reward)**：用户执行建议 -> 焦虑缓解 -> 生成“水逆退散符”壁纸并分享到朋友圈（社交酬赏）。

+## 3. 系统功能架构设计 (Functional Architecture)
+
+### 3.1 前台 (Client-Side): 沉浸式数字圣所
+*   **Dashboard (Bento Grid布局)**：
+    *   **今日能量卡**：GenAI生成的动态3D图腾（视觉化运势）。
+    *   **行动指南 (Actionable Nudges)**：基于运势的To-Do List（如“宜：整理桌面”，“忌：冲动消费”）。
+    *   **实时状态机**：类似股市K线的人生能量波动图（Life K-Line）。
+*   **仪式引擎 (Ritual Engine)**：
+    *   **触感六爻**：利用手机陀螺仪与线性马达，模拟真实的摇卦手感。
+    *   **电子木鱼/颂钵**：结合呼吸节奏的解压工具。
+*   **关系实验室 (Social Lab)**：
+    *   **幽灵档案 (Ghost Profiles)**：允许单方面输入他人信息进行合盘。
+    *   **关系剧本**：AI模拟双方对话，预测冲突点。
+
+### 3.2 中台 (Business Logic Layer): 混合智能核心
+*   **确定性计算引擎 (Deterministic Core)**：
+    *   **天文算法库**：基于 `Swiss Ephemeris` 和 `LunarDate`，负责真太阳时校正、八字排盘、紫微星盘计算。（*原则：数学问题绝不用LLM猜*）
+*   **推理增强引擎 (RAG + Knowledge Graph)**：
+    *   **命理知识图谱**：存储“十神生克”、“星曜组合”的结构化规则。
+    *   **向量数据库**：存储古籍（滴天髓等）与现代心理学话术。
+*   **Agentic AI 服务**：
+    *   **人格路由器**：根据用户偏好切换“毒舌导师”或“温柔疗愈师”模式。
+    *   **记忆系统**：记录用户的情绪日志与历史事件，实现“长期记忆”。
+*   **AIGC 渲染服务**：
+    *   **Prompt 编排器**：将命理参数（如“火旺”）转译为视觉Prompt（如“Neon Fire, Cyberpunk style”）。
+
+### 3.3 后台 (Admin/Operations): 合规与洞察
+*   **合规风控系统**：自动扫描所有AI生成内容，拦截“封建迷信”、“绝对宿命论”（如“必死”、“血光”）词汇，替换为心理学建议。
+*   **用户情绪洞察**：分析全平台用户的情绪热力图，辅助运营活动（如全网焦虑时推送疗愈内容）。

+## 4. 关键业务流程设计 (Key Business Flows)
+
+### 4.1 场景一：主动式危机干预 (The "Cosmic Watchtower" Flow)
+*此流程旨在自动化处理真人大师的“流年预警”服务。*
+
+1.  **数据监控**：后台批处理任务每日凌晨4点运行，计算用户本命盘与当日行运星盘（Transit）的相位关系。
+2.  **规则触发**：检测到高风险相位（如：流年土星冲本命月亮 -> 对应“情绪压抑/家庭变故”）。
+3.  **AI推理**：
+    *   调用 RAG 获取该相位的心理学解释。
+    *   结合用户历史日记（如最近常提“失眠”），生成个性化建议。
+4.  **主动触达**：APP推送消息：“最近感到压力大吗？土星正在考验你的内心。点击获取‘情绪急救包’。”
+5.  **服务闭环**：用户点击 -> 进入“情绪疗愈”页面 -> 引导进行5分钟冥想（音频） -> 推荐“安神壁纸”。
+
+### 4.2 场景二：社交合盘与裂变 (The "Social Synastry" Flow)
+*此流程旨在利用“窥私欲”实现低成本获客。*
+
+1.  **输入**：用户A输入暗恋对象B的生日（无需B注册）。
+2.  **计算**：系统生成“单向合盘报告”。
+    *   *差异化点*：不只给分数，而是给**场景**。例如：“你们在‘吃喝玩乐’上是满分搭档（食神互通），但在‘搞钱’上可能会吵架（比劫夺财）。”
+3.  **诱导**：报告模糊关键细节（如“B内心最真实的想法”），提示“邀请B解锁完整双向视角”。
+4.  **裂变**：生成一张带有二维码的“关系能量图”，A分享给B -> B扫描进入，成为新用户。

+## 5. 技术架构原则 (Technical Architecture Principles)
+
+### 5.1 系统稳定性与扩展性
+*   **混合架构 (Hybrid Architecture)**：
+    *   **Local-First (端侧优先)**：仪式类功能（木鱼、摇卦）必须在本地运行，利用端侧计算保证零延迟的触觉反馈（Haptics），确保沉浸感。
+    *   **异步计算 (Asynchronous Processing)**：复杂的全盘推演和图像生成（Nano Banana）放入消息队列（Kafka/RabbitMQ），避免阻塞主线程。
+*   **结构化 RAG (Structured RAG)**：
+    *   针对 `survey real expert.md` 中提到的AI幻觉问题，我们不直接将古籍丢给LLM。而是先通过规则引擎提取结构化结论（如“格局：伤官配印”），再将此结论作为 Context 喂给 LLM 进行润色。**“数学脑”管对错，“语言脑”管温度。**
+
+### 5.2 数据隐私与安全
+*   **数字告解室标准 (Digital Confessional Standard)**：
+    *   **PII 隔离**：用户的生辰八字和真实姓名在存储时加密，且与对话日志物理隔离。
+    *   **LLM 脱敏**：向第三方 LLM 发送请求时，剥离所有身份信息，仅发送命理参数（如“User_123: Male, 1995-05-01” -> “Profile A: Fire Day Master”）。
+    *   **遗忘权**：提供“一键焚毁”功能，彻底删除所有命盘记录和对话历史，模拟“烧掉信件”的仪式感。

+## 6. 差异化竞争壁垒 (The Moat)
+
+### 6.1 为什么用户无法回到竞品？
+*   **比真人更即时**：真人大师需要预约、排队、付费。DaoMind 的“宇宙瞭望塔”系统是 24/7 实时监控，在用户意识到问题之前就给出了预警。
+*   **比竞品更懂“人情世故”**：大多数 AI 算命只会背书。我们的 Agentic AI 结合了中国社会的语境（基于 `survey real expert.md` 的洞察），懂得用“犯小人”来解释职场政治，用“桃花劫”来解释情感纠纷，提供了符合文化心理的解释框架。
+*   **无法割舍的“数字资产”**：用户在系统中积累的不仅仅是运势记录，而是**“情绪日记”**与**“能量图腾”**。当系统准确记录并解释了用户过去一年的心路历程（Hindsight Validation），用户将产生极高的迁移成本。
+
+### 6.2 总结
+DaoMind 的核心壁垒在于**将“玄学”重构为“数据科学”与“艺术疗愈”的结合体**。我们不贩卖焦虑，我们贩卖的是不确定时代下的**掌控感**与**美学体验**。
