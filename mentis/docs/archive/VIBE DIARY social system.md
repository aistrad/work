# 第一部分：系统概述

## 1.1 核心设计理念

```
╔═══════════════════════════════════════════════════════════════════════════════╗
║                                                                               ║
║   VIBE INSIGHT MODULE SYSTEM                                                  ║
║   让 Vibe 成为连接用户与专业智慧的桥梁                                         ║
║                                                                               ║
║   ═══════════════════════════════════════════════════════════════════════     ║
║                                                                               ║
║   核心原则：                                                                   ║
║                                                                               ║
║   1. VIBE 是唯一入口                                                          ║
║      用户永远和 Vibe 对话，模块通过 Vibe 被"调用"                              ║
║                                                                               ║
║   2. VIBE 不扮演专家                                                          ║
║      Vibe 负责理解、连接、协调、总结                                           ║
║      专业内容由 Insight Module 提供                                            ║
║                                                                               ║
║   3. 订阅决定能力                                                             ║
║      默认模块 + 订阅模块 = Vibe 的"内化能力"                                   ║
║      未订阅模块 = 需要确认后调用                                              ║
║                                                                               ║
║   4. 模块卡片化                                                               ║
║      模块以独立卡片形式呈现                                                    ║
║      输入、流程、输出都在卡片内完成                                            ║
║                                                                               ║
╚═══════════════════════════════════════════════════════════════════════════════╝
```

## 1.2 三种调用模式

```
┌─────────────────────────────────────────────────────────────────────────┐
│                                                                         │
│   INSIGHT MODULE 三种调用模式                                           │
│                                                                         │
│   ═══════════════════════════════════════════════════════════════════   │
│                                                                         │
│   MODE 1: 隐式调用（Vibe 的内化能力）                                   │
│   ─────────────────────────────────────────────────────────────────     │
│   适用范围：默认模块 + 用户已订阅模块                                   │
│   用户感知：Vibe 本来就懂这个领域                                       │
│   Vibe 行为：直接回答，不提及来源，不需确认                             │
│                                                                         │
│   ═══════════════════════════════════════════════════════════════════   │
│                                                                         │
│   MODE 2: 模块推荐（未订阅模块）                                        │
│   ─────────────────────────────────────────────────────────────────     │
│                                                                         │
│   ┌─────────────────────────────────────────────────────────────────┐   │
│   │                                                                 │   │
│   │   Mode 2a: 显式请求                                             │   │
│   │   ─────────────────────────────────────────────────────────     │   │
│   │   用户明确请求某类专业服务                                      │   │
│   │   例："帮我看看八字" "分析一下我的依恋模式"                     │   │
│   │                                                                 │   │
│   │   流程：直接确认 → 弹出模块卡片 → 卡片内完成全流程              │   │
│   │                                                                 │   │
│   │   ─────────────────────────────────────────────────────────     │   │
│   │                                                                 │   │
│   │   Mode 2b: 隐含意图                                             │   │
│   │   ─────────────────────────────────────────────────────────     │   │
│   │   用户问题隐含专业需求，但未明确请求                            │   │
│   │   例："我有个创业想法" "最近总是焦虑"                           │   │
│   │                                                                 │   │
│   │   流程：粗浅回答 + 推荐模块 → 确认 → 弹出模块卡片               │   │
│   │                                                                 │   │
│   └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│   ═══════════════════════════════════════════════════════════════════   │
│                                                                         │
│   MODE 3: 深度对话（三方/多方对话）                                     │
│   ─────────────────────────────────────────────────────────────────     │
│   适用范围：用户主动请求 / 复杂问题需要系统探索                         │
│   用户感知：和专家直接对话，Vibe 作为协调者陪伴                         │
│   角色分工：                                                            │
│     • 专家：主导专业对话                                                │
│     • Vibe：提供背景、协调、翻译、总结                                  │
│     • 可引入多个专家进行跨领域探讨                                      │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 4.2 调用决策流程

```
┌─────────────────────────────────────────────────────────────────────────┐
│                                                                         │
│   MODULE ROUTING DECISION FLOW                                          │
│                                                                         │
│   ═══════════════════════════════════════════════════════════════════   │
│                                                                         │
│   function routeToModule(message, context, user) {                      │
│                                                                         │
│     // Step 1: 检测深度对话请求                                         │
│     if (isDeepDialogueRequest(message)) {                               │
│       return { mode: 'MODE_3', ... }                                    │
│     }                                                                   │
│                                                                         │
│     // Step 2: 识别意图类型                                             │
│     const intentType = analyzeIntentType(message)                       │
│     // 'explicit' | 'implicit' | 'none'                                │
│                                                                         │
│     if (intentType === 'none') {                                        │
│       return { mode: 'VIBE_ONLY' }                                      │
│     }                                                                   │
│                                                                         │
│     // Step 3: 搜索匹配相关模块                                         │
│     const matchedModules = searchModules(message, context)              │
│                                                                         │
│     if (matchedModules.length === 0) {                                  │
│       return { mode: 'VIBE_ONLY' }                                      │
│     }                                                                   │
│                                                                         │
│     // Step 4: 检查订阅状态                                             │
│     const bestMatch = matchedModules[0]                                 │
│     const isSubscribed = user.subscriptions.includes(bestMatch.id)      │
│                          || bestMatch.category === 'default'            │
│                                                                         │
│     // Step 5: 决定调用模式                                             │
│     if (isSubscribed) {                                                 │
│       return { mode: 'MODE_1', module: bestMatch }                      │
│     }                                                                   │
│                                                                         │
│     if (intentType === 'explicit') {                                    │
│       return { mode: 'MODE_2a', module: bestMatch }                     │
│     }                                                                   │
│                                                                         │
│     return { mode: 'MODE_2b', module: bestMatch }                       │
│   }                                                                     │
│                                                                         │
│   // 显式请求关键词                                                     │
│   const explicitKeywords = [                                            │
│     "帮我看看", "帮我分析", "帮我测试",                                 │
│     "做个分析", "做个测试", "给我算算",                                 │
│     "分析一下我的", "看看我的"                                          │
│   ]                                                                     │
│                                                                         │
│   function analyzeIntentType(message) {                                 │
│     if (explicitKeywords.some(k => message.includes(k))) {              │
│       return 'explicit'                                                 │
│     }                                                                   │
│     if (hasImplicitModuleNeed(message)) {                               │
│       return 'implicit'                                                 │
│     }                                                                   │
│     return 'none'                                                       │
│   }                                                                     │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```
