# 合婚配对

## 元数据

- **ID**: compatibility
- **名称**: 合婚配对
- **级别**: professional
- **计费**: premium
- **描述**: 分析双方命盘的匹配度，给出相处建议

## 触发条件

**主要触发词**: 合婚, 配对, 合不合, 般配

**次要触发词**: 两个人合适吗, 能在一起吗

## 前置要求

- 双方出生信息（必须）
- 双方性别（必须）

## 输出配置

**使用工具**:
- show_bazi_chart (双方)
- show_insight

**交付物**: 合婚分析报告

**预估轮次**: 5-10 轮

## 服务流程 (SOP)

### Phase 1: 信息收集

**类型**: required

**描述**: 收集双方出生信息

**收集内容**:
| 信息 | 必填 | 说明 |
|-----|------|------|
| 甲方出生信息 | ✅ | 年月日时 |
| 甲方性别 | ✅ | 男/女 |
| 乙方出生信息 | ✅ | 年月日时 |
| 乙方性别 | ✅ | 男/女 |

### Phase 2: 计算排盘

**类型**: required

**描述**: 分别计算双方命盘

### Phase 3: 快速扫描

**类型**: required

**描述**: 快速识别匹配特征

**扫描重点**:
1. 日柱关系（合/冲/刑/害）
2. 年柱关系
3. 五行互补情况
4. 配偶星与对方日主关系

**合婚速判**:
| 特征 | 匹配度 |
|-----|--------|
| 日柱相合 | 高度匹配 |
| 五行互补 | 良好匹配 |
| 日柱相冲 | 需要磨合 |
| 五行相克 | 挑战较多 |

### Phase 4: 主动追问

**类型**: optional

**描述**: 了解双方关系状况

**追问示例**:
- "你们目前是什么关系？"
- "在一起多久了？"
- "主要的分歧是什么？"

### Phase 5: 深度分析

**类型**: required

**分析框架**:

1. **五行互补分析**
   - 双方五行配置
   - 互补还是相克
   - 平衡建议

2. **性格匹配分析**
   - 双方性格特点
   - 相处模式预测
   - 潜在冲突点

3. **配偶星分析**
   - 甲方配偶星与乙方匹配度
   - 乙方配偶星与甲方匹配度

4. **婚姻宫分析**
   - 双方日支关系
   - 婚姻稳定性

5. **相处建议**
   - 如何发挥优势
   - 如何化解冲突
   - 注意事项

### Phase 6: 输出交付

**类型**: required

**工具调用**:
1. `show_bazi_chart` - 展示甲方命盘
2. `show_bazi_chart` - 展示乙方命盘
3. `show_insight(type="compatibility")` - 合婚洞察

**输出结构**:
```
【甲方命盘】
{调用 show_bazi_chart}

【乙方命盘】
{调用 show_bazi_chart}

【合婚分析】
{调用 show_insight}

## 匹配度评估

**整体匹配度**: {评分}/100

**优势互补**:
- {互补点1}
- {互补点2}

**需要注意**:
- {注意点1}
- {注意点2}

## 相处建议

{具体建议}
```

### Phase 7: 答疑讨论

**类型**: reactive

**常见问题**:
| 问题 | 处理 |
|-----|------|
| "能结婚吗" | 综合分析给建议，不做绝对判断 |
| "会离婚吗" | 委婉分析风险点，给出化解建议 |
| "怎么相处" | 给出具体可操作的建议 |
