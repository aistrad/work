# 基础解读

## 元数据

- **ID**: basic_reading
- **名称**: 基础解读
- **级别**: entry
- **计费**: free
- **描述**: 快速解读八字命盘的核心特征，帮助用户了解自己的基本命理信息

## 触发条件

**主要触发词**: 看命盘, 算命, 测算, 帮我看看, 分析一下

**次要触发词**: 八字, 命理, 生辰

## 前置要求

- 用户出生日期（必须）
- 用户出生时间（必须）
- 出生地点（可选，用于真太阳时校正）

## 输出配置

**使用工具**:
- show_bazi_chart
- show_insight

**交付物**: 命盘图 + 核心特征洞察

**预估轮次**: 1-3 轮

## 服务流程 (SOP)

### Phase 1: 信息收集

**类型**: required

**描述**: 检查并收集用户出生信息

**检查点**: 用户是否已有 birth_info？
- **有 birth_info** → 跳过收集，直接进入 Phase 2
- **无 birth_info** → 调用 `collect_bazi_info` 收集

**收集内容**:
| 信息 | 必填 | 说明 |
|-----|------|------|
| 出生日期 | ✅ | 年月日 |
| 出生时间 | ✅ | 时辰 |
| 出生地点 | ⚪ | 用于真太阳时校正 |

### Phase 2: 计算排盘

**类型**: required

**描述**: 调用计算工具生成命盘数据

**工具调用**:
```
calculate_bazi:
  include_dayun: true
```

### Phase 3: 快速扫描

**类型**: required

**描述**: 30秒内完成核心模式识别

**执行内容**:
1. 日主强弱判断
2. 格局快速识别
3. 红旗信号检查

**判断规则**:
| 信号 | 判断 |
|-----|------|
| 日主在月令得时 + 地支有根 | 身强 |
| 日主在月令失时 + 地支无根 | 身弱 |
| 比劫成群 | 身强（即使月令不旺） |
| 官杀重重 + 日主无根 | 身弱受克 |

### Phase 4: 主动追问

**类型**: optional

**描述**: 根据命盘特点，可以自然地追问用户以提供更精准分析

**触发条件**: 当检测到需要更多信息才能给出准确分析时

**追问示例**:
| 命盘特点 | 追问问题 |
|---------|---------|
| 官杀混杂 | "你目前的工作稳定吗？" |
| 桃花旺 | "你目前的感情状态如何？" |
| 财星受克 | "最近有投资计划吗？" |

**用户可选择回答或跳过**

### Phase 5: 深度分析

**类型**: required

**描述**: 结合命盘数据进行分析

**分析重点**:
- 日主特性（性格基调）
- 格局高低（人生格局）
- 用神喜忌（发展方向）

### Phase 6: 输出交付

**类型**: required

**描述**: 调用展示工具呈现结果

**工具调用顺序**:
1. `show_bazi_chart` - 展示命盘图
2. `show_insight` - 展示核心洞察（不超过3点）

**输出结构**:
1. 先调用展示工具
2. 再用简短文字说明核心观点
3. 预判用户可能的追问

### Phase 7: 答疑讨论

**类型**: reactive

**描述**: 回答用户追问

**常见问题处理**:
| 问题模式 | 处理方式 |
|---------|---------|
| "什么意思" | 用更通俗的语言解释 |
| "准不准" | 承认命理是参考，鼓励验证 |
| "怎么化解" | 给出具体可操作的建议 |

## 输出模板

